# ADR-008 · Organization-as-tenant; multi-tenant-ready, single-tenant-first

**Status:** accepted · **Date:** 2026-07-02 (pre-existing)

## Контекст
Нужно расшириться в Фазу 2 без переписывания модели данных.

## Решение
Иерархия `organization → subdivision → membership(user↔subdivision+role)`; `users` глобальны. `organization_id` денормализован в каждую операционную строку (`ondelete=RESTRICT`); tenant-репозитории требуют scope в конструкторе; scope берётся только из подписанного JWT. В Фазе 1 используется единственный default-tenant.

## Последствия
Жёсткая изоляция с первого дня при минимальной реализации.
