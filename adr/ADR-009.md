# ADR-009 · One implementation per provider-seam until a real trigger

**Status:** accepted (с активным исключением) · **Date:** 2026-07-02 (pre-existing)

## Контекст
Мёртвый код «на будущее» — риск распыления усилий.

## Решение
Интерфейсы (OCR/ERP за `Protocol` + registry) существуют, но активна ровно одна реализация: OCR=`claude`, ERP=`esupl`. Альтернативы не пишутся без триггера (рост scope, географический/стоимостной риск).

## Последствия
Транспорт LLM сейчас — модульные функции, а не `Protocol`.

## Исключение / долг
Зарегистрирован второй OCR/AI-вендор `gemini` — это нарушает решение. Требует резолюции: claude-only (рекомендуется) или завершение seam. → **BACKLOG DEC-01**.

## Обновление 2026-07-21 (append-only)
Для **ERP-шва** это решение частично вытеснено [[ADR-024]]: multi-POS-триггер сработал, ERP несёт две санкционированные реализации (`esupl` + `quickresto`, выбор пер-организацию). Инвариант «одна реализация на **OCR**-шов» (OCR=`claude`) остаётся в силе; долг второго OCR-вендора gemini (DEC-01) — без изменений.
