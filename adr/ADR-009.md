# ADR-009 · One implementation per provider-seam until a real trigger

**Status:** accepted (with an active exception) · **Date:** 2026-07-02 (pre-existing)

## Context
Dead code "for the future" is a risk of spreading effort thin.

## Decision
Interfaces (OCR/ERP behind a `Protocol` + registry) exist, but exactly one implementation is active: OCR=`claude`, ERP=`esupl`. Alternatives are not written without a trigger (scope growth, geographic/cost risk).

## Consequences
The LLM transport is currently modular functions, not a `Protocol`.

## Exception / debt
A second OCR/AI vendor, `gemini`, is registered — this violates the decision. It requires a resolution: claude-only (recommended) or completing the seam. → **BACKLOG DEC-01**.
