# ADR-005 · Three-tier separation of configuration/secrets, no env fallback

**Status:** accepted · **Date:** 2026-07-02 (pre-existing, config-arch-review)

## Context
Secrets in code/env and the need to redeploy just to change a key.

## Decision
Three tiers.
1. `.env` — boot/trust-root only (DB, KEK, JWT/session, admin operator, static ERP selection, URLs, cookie flags, VPN declarative).
2. `system_settings` (DB) — non-secret runtime settings.
3. `integration_credentials` (DB, encrypted) — integration secrets.

For tiers (2)/(3) there is **no fallback to env**.

## Consequences
Every value has exactly one source; AI keys/toggles/POS token moved out of env into the DB, managed via the super-admin console.
