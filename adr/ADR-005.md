# ADR-005 · Трёхуровневое разделение конфигурации/секретов, без фолбэка в env

**Status:** accepted · **Date:** 2026-07-02 (pre-existing, config-arch-review)

## Контекст
Секреты в коде/env и необходимость передеплоя лишь ради смены ключа.

## Решение
Три уровня.
1. `.env` — только boot/trust-root (DB, KEK, JWT/session, admin-оператор, статический выбор ERP, URL-ы, cookie-флаги, VPN декларативно).
2. `system_settings` (DB) — несекретные runtime-настройки.
3. `integration_credentials` (DB, зашифрованные) — секреты интеграций.

Для уровней (2)/(3) **нет фолбэка в env**.

## Последствия
У каждого значения ровно один источник; AI-ключи/тумблеры/POS-токен вынесены из env в DB, управляются через консоль супер-админа.
