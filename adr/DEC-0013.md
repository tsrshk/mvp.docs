# DEC-0013 · Commit требует ПОДТВЕРЖДЁННОЙ идентичности SKU (вариант A)

**Status:** Accepted · **Date:** 2026-07-08 · **Codified in:** [[ADR-018]]
**Decided by:** Ivan (окно вето TZ `TZ__DEC-0011_merge-gate_2026-07-08.md`)
**Depends on:** [[DEC-0011]] (POS как SoT идентичности; two-phase authority)
**Relates to:** VER-021 (durability, GATE, открыт), VER-022 (scope кэша, закрыт этим решением)

## Контекст
Строка без подтверждённого `sku_mapping`, но с exact-match в `ingredient_cache` (Tier 3) — коммитить её автоматически или нет? TZ предлагало **вариант C** по умолчанию (auto-resolve exact-match + auto-create mapping). Открытый вопрос §3.4 патча.

## Решение — вариант A (block until manual confirmation)
**Commit разрешён ТОЛЬКО для подтверждённой идентичности.** Строка без commit-eligible `sku_mapping` (`method=manual` **ИЛИ** `confirmed_by IS NOT NULL`) при commit → **block + review**, не запись. Exact-match `ingredient_cache` и fuzzy/AI — хинты draft-фазы; они **не** автокоммитятся и **не** автосоздают маппинг. Moat растёт через явное подтверждение человеком (создание ручного маппинга или подтверждение хинта → `confirmed_by`).

Это строже рекомендации патча (§3.4) и строже варианта C: наиболее fail-closed прочтение принципа «накладная — ответственная стадия; AI показывает, человек решает».

### Рассмотренные варианты
| Variant | UX friction | Moat | Fail-closed | Choice |
|---|---|---|---|---|
| A. Block until manual confirmation | high (на каждом новом элементе) | yes (через confirmation) | yes | **✅ accepted** |
| B. Auto-resolve exact without mapping | none | no | yes | rejected |
| C. Auto-resolve exact + auto-mapping | none | yes | yes | rejected (был default TZ) |

> **Вето варианта C подтверждено 2026-07-09** ([[ADR-018]]): попытка `TZ__STABILIZATION_2026-07-09` (D1/S6) возродить C отклонена — она ломает merge_gate тест `test_exact_cache_match_does_not_commit_and_creates_no_mapping`.

## Нормативные правила

**MUST**
- Commit-резолюция: `normalize_source_key(line.description)` → `sku_mapping`, приоритет subdivision → org.
- Commit-eligible = `method == manual` OR `confirmed_by IS NOT NULL`.
- Нет commit-eligible маппинга → block+review (статус `rejected`), приход НЕ записывается, валидация НЕ пропускается.

**MUST NOT**
- Cache exact-match / fuzzy / AI MUST NOT автокоммитить и MUST NOT автосоздавать `sku_mapping`.
- Commit-путь MUST NOT читать `ingredient_cache` как источник авторитета (согласованно с [[DEC-0011]]).

## Последствия
- (+) Ни одна неподтверждённая догадка не доходит до POS. Откат не нужен — мусор не пишется.
- (+) VER-022 закрыт: cache вне commit-пути → асимметрия scope «cache is org-only» невозможна.
- (−) Каждый новый элемент требует единоразового подтверждения (ожидаемое трение; это и есть «review»).
- Реализация: `invoice_service._resolve_commit_identities`; тесты `test_dec0013_commit_gate.py` (`test_exact_cache_match_does_not_commit_and_creates_no_mapping`, `test_fuzzy_mapping_without_confirmation_blocks_commit`, `test_confirmed_mapping_commits`) — merge_gate.
