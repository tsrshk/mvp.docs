# ADR-006 · Fail-closed повсюду

**Status:** accepted · **Date:** 2026-07-02 (pre-existing)

## Контекст
Тихая деградация опаснее явного отказа.

## Решение
Отсутствие/недоступность зависимости → жёсткая ошибка, а не деградация:
- VPN недоступен (при `ai_vpn_enabled`) → `VpnUnavailableError`, никакого тихого прямого egress.
- Нет AI-ключа → `AiUnavailableError`.
- Нет POS-токена → Esupl 401.
- Шифртекст при пустом keyring → `RuntimeError`.
- `erp_write_enabled` по умолчанию `False`.

## Последствия
Никаких «резервных путей»; предсказуемость важнее доступности.
