# ADR-004 · Esupl as the primary ERP

**Status:** accepted · **Date:** 2026-07-02 (pre-existing)

## Context
Customer Zero runs on the Belarusian POS Esupl.

## Decision
The primary (and, in Phase 1, the only) ERP is Esupl. `organization ↔ exactly one Esupl team`; `subdivision ↔ Esupl warehouse`. iiko and others — Phase 2.

## Consequences
The ERP provider is selected statically (`ERP_PROVIDER` from env); a seam for other ERPs exists but is not implemented until a trigger.
