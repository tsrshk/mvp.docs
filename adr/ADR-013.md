# ADR-013 · Photo-first invoice flow

**Status:** accepted · **Date:** 2026-06-29 (recognition-feature)

## Context
The supplier is unknown until recognition.

## Decision
The supplier is not selected in advance — OCR reads the name from the photo and the system matches it against the POS list (`bestNameMatch`, ≥0.6 auto, otherwise uncertain + dropdown). supplier→SKU+packing mappings are remembered and auto-applied on the next invoice from the same supplier (learning loop).

> **Code clarification (2026-07-09):** the supplier match uses a blended score — trigram `0.65` + token-Jaccard `0.35`, threshold `min 0.4` (NOT "Jaccard ≥ 0.5" as earlier docs stated). The supplier auto-accept threshold (`≥0.6`) applies to `bestNameMatch` and is retained. See [[erp-esupl-integration]] / [[supplier-criteria-registry]].

## Consequences
The supplier is committed in the same dispatch that reads its id.

> **Historical note:** ADR-013 was sometimes cited as the source of the "moat accumulation channel / persist survives reject" decision. This is incorrect — that decision is codified separately in [[ADR-020]]; the dangling reference from APP_OVERVIEW §8 has been removed.
