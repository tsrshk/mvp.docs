# ADR-014 · OCR via a vision-LLM, not a classic OCR engine

**Status:** accepted · **Date:** 2026-06-22 (scan-preprocessing-plan)

## Context
The choice of image preprocessing pipeline.

## Decision
Recognition is done by a vision-LLM (Claude Vision; Gemini was considered). Therefore **no** binarization/thresholding/hard grayscale (these hurt the VLM). Preprocessing: silent normalization (EXIF, downscale ≤1568px, a single JPEG pass, fail-open), crop is "auto-first, manual-optional", not mandatory.

## Consequences
No UI for manual field markup; the bet is on tight-crop + resize + confidence-gate.
