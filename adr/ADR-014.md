# ADR-014 · OCR через vision-LLM, а не классический OCR-движок

**Status:** accepted · **Date:** 2026-06-22 (scan-preprocessing-plan)

## Контекст
Выбор пайплайна предобработки изображения.

## Решение
Распознавание выполняет vision-LLM (Claude Vision; рассматривался Gemini). Поэтому **никакой** бинаризации/thresholding/жёсткого grayscale (это вредит VLM). Предобработка: тихая нормализация (EXIF, downscale ≤1568px, единственный проход JPEG, fail-open), crop — «auto-first, manual-optional», не обязателен.

## Последствия
Нет UI для ручной разметки полей; ставка на tight-crop + resize + confidence-gate.
