# ADR-007 · Две независимые плоскости аутентификации

**Status:** accepted · **Date:** 2026-07-02 (pre-existing, CLAUDE.md §11)

## Контекст
Смешение admin- и user-аутентификации = риск эскалации привилегий.

## Решение
Две раздельные плоскости, которые не смешиваются:
- **App-пользователи** (`users`, argon2, JWT access + opaque refresh, cookies).
- **SQLAdmin-оператор** (env `ADMIN_USERNAME` + bcrypt, session cookie, **без строки в `users`**).

## Последствия
Два набора hash/verify; оператор не заводится в `users`. Маршруты `admin_system` гейтятся плоскостью SQLAdmin OPERATOR, а не app-JWT супер-админом.
