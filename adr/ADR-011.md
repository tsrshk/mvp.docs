# ADR-011 · Secrets are read without caching

**Status:** accepted · **Date:** 2026-07-02 (pre-existing)

## Context
Caching a secret makes rotation indistinguishable from a bug.

## Decision
`get_active_credential` reads + decrypts on every call (no cache) — rotation in the super-admin console is instant. One active secret per `(scope, provider, org, subdivision)` — a partial-unique index.

## Consequences
The extra DB query is accepted as the price of determinism.
