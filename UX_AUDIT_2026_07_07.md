# UX Аудит: Invoice Form Page
**Дата:** 2026-07-07  
**Статус:** Критические проблемы выявлены  
**Фокус:** Scrolling behavior, content accessibility, mobile responsiveness

---

## 📋 Executive Summary

Страница формы накладной имеет **7 критических и высокоприоритетных проблем**, блокирующих нормальное использование, особенно на мобильных устройствах:
- **Нет прокрутки таблицы** - содержимое полностью за пределами экрана
- **Фиксированная полоса закрывает контент** - кнопка "Добавить" невидима
- **Поля не адаптируются к мобилю** - требуют горизонтального скролла
- **Touch targets слишком маленькие** - ниже 44x44px стандарта

**Общая оценка:** Плохой UX, требует срочных исправлений

## 🔴 КРИТИЧЕСКИЕ ПРОБЛЕМЫ (P0)

### 1. Нет прокрутки контента - ГЛАВНАЯ ПРОБЛЕМА
**Файл:** `InvoiceImportPage.tsx`, `InvoiceWorkbench.tsx`  
**Строки:** MAIN_PAGE_CONTAINER, root div layout  
**Проблема:**
- MAIN_PAGE_CONTAINER не имеет явной высоты (`max-w-[1340px]` только max-width)
- Весь контент течёт вниз одним потоком без ограничений
- На мобиле таблица полностью за пределами видимого экрана
- На десктопе таблица обрезана внизу, нет видимого скроллбара

**Видимое поведение:**
- Мобиль: после photo accordion видна только первая строка таблицы
- Десктоп: таблица обрезана, пользователь не может видеть/скроллить товары

**Причина:**
```
<div className={MAIN_PAGE_CONTAINER}>
  {/* Breadcrumbs */}
  {/* Stepper */}
  {/* Form fields */}
  {/* Photo */}
  {/* Photo accordion + table */}
  {/* Footer */}
</div>
```
Вся структура - поток без явного управления высотой

**Решение:**
- Добавить `h-screen` и `overflow-auto` на MAIN_PAGE_CONTAINER
- Или создать separate scrollable zone для content
- Мобиль: использовать viewport height с компенсацией на header/footer

**Impact:** 🔴 **КРИТИЧНО** - пользователь не может видеть таблицу товаров

---

### 2. Фиксированная полоса действий закрывает контент
**Файл:** `InvoiceWorkbench.tsx`  
**Строки:** 566-573 (mobile action bar)  
**Проблема:**
```html
<div className="fixed inset-x-0 bottom-0 z-30 ... pb-[max(0.625rem,env(safe-area-inset-bottom))]">
  <!-- Кнопки, статус -->
</div>
```
- Mobile action bar имеет `fixed` позицию, z-30
- **Нет компенсирующего padding в workbench**
- Кнопка "Добавить строку" (add-row button) скрывается за полосой

**Видимое поведение:**
- На мобиле последняя карточка товара скрыта
- Кнопка "+ Добавить строку" невидима
- Пользователь не может добавить новую строку

**Решение:**
- В InvoiceWorkbench добавить `pb-[80px]` или динамический отступ
- Или использовать `mb-safe` CSS переменную
- Пример: `<div className="pb-[calc(80px_+_env(safe-area-inset-bottom))]">`

**Impact:** 🔴 **КРИТИЧНО** - одна из основных операций (добавить товар) недоступна

---

### 3. Фото viewer обрезает изображение без прокрутки
**Файл:** `InvoiceWorkbench.tsx`  
**Строки:** 519-523  
**Проблема:**
```html
<div className="h-[20vh] w-full rounded-[10px] overflow-hidden">
  <PhotoViewer />
</div>
```
- Высота зафиксирована на `h-[20vh]`
- На мобилях < 500px это становится 80-100px (очень мало)
- `overflow-hidden` блокирует прокрутку/панорамирование
- Пользователь не может рассмотреть детали накладной

**Видимое поведение:**
- Мобиль (320px): фото только ~80px высоты
- Невозможно разглядеть номер накладной, детали
- Нет возможности масштабировать/прокручивать фото

**Решение:**
- На мобиле: `h-[40vh]` или `h-[300px]` min-height
- Сделать scrollable: `overflow-auto` вместо `overflow-hidden`
- Добавить touch-zoom (pinch) или allow horizontal scroll для wide images

**Impact:** 🔴 **КРИТИЧНО** - нельзя разглядеть исходный документ

## 🟡 ВЫСОКОПРИОРИТЕТНЫЕ ПРОБЛЕМЫ (P1)

### 4. Поля формы не адаптируются к мобилю
**Файл:** `InvoiceWorkbench.tsx`  
**Строки:** 347-483 (header fields section)  
**Проблемы:**

#### a) Поставщик имеет max-width
```
<select className={`${H_INPUT} max-w-[200px] ...`}>
```
- max-w-[200px] = 200px, на мобиле 320px это 62.5% ширины
- Длинные названия поставщиков обрезаются
- Нельзя видеть полное имя

#### b) Серия накладной слишком узкая
```
<input className={`${H_INPUT} w-14 ...`} placeholder="ЕМ" />
```
- w-14 = 56px для ввода "ЕМ" (2 символа)
- На мобиле неудобно вводить

#### c) Сумма слишком узкая для decimals
```
<input className={`${H_INPUT} w-24 ...`} /> + <span>{currency}</span>
```
- w-24 = 96px для числа + валюта
- На мобиле очень узко

#### d) Весь header использует flex-wrap
```
<div className="flex flex-wrap items-center gap-x-[18px] gap-y-2">
```
- На мобиле создаёт 4-5 строк разной высоты
- Сложно сканировать взглядом
- Непредсказуемые переносы

**Видимое поведение:**
```
Мобиль (320px):
[⚠ Поставщик: [ООО Велес Ф...]]
[Дата: [07/07]] [Склад: [Основной...]]
[Накладная: [ЕМ] [4920500]]
[Итог: [188.78] [RUB]]
```
Сложно читать, не очевидна структура

**Решение:**
- На мобиле (< 640px): каждое поле на новой строе, `w-full`
- Desktop (≥ 640px): flex-wrap OK
- Вертикальный стэк с clear structure

**Code fix:**
```html
<!-- На мобиле -->
<div className="flex flex-col gap-3 md:flex-row md:flex-wrap ...">
  <label className="w-full md:w-auto">...</label>
  <label className="w-full md:w-auto">...</label>
</div>
```

**Impact:** 🟡 **ВЫСОКОЕ** - 80%+ пользователей на мобиле испытывают friction

### 5. LineCard grid-cols-3 на всех экранах
**Файл:** `LineCard.tsx`  
**Строки:** 100  
**Проблема:**
```html
<div className="mt-3 grid grid-cols-3 gap-2">
  <Field label="Кол-во">...</Field>
  <Field label="Ед.">...</Field>
  <Field label="Фасовка">...</Field>
  <Field label="Цена/ед">...</Field>
  <!-- ... -->
</div>
```
- grid-cols-3 на всех размерах экрана
- На 320px: каждая колонка только ~86px ширины
- На 360px: каждая колонка ~106px (всё ещё узко)
- Невозможно вводить числа в такие узкие поля

**Видимое поведение:**
- Мобиль: 3 поля рядом в очень узких столбцах
- Текст обрезается, численные поля не читаются
- Пользователю сложно вводить данные

**Решение:**
- На мобиле (< 640px): `grid-cols-1` (single column)
- На tablet (640-1024px): `grid-cols-2` (two columns)
- На десктопе (≥ 1024px): `grid-cols-3` (три колонки)

**Code fix:**
```html
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-2">
```

**Impact:** 🟡 **ВЫСОКОЕ** - пользователи на мобилях не могут комфортно вводить данные

### 6. Touch targets слишком маленькие
**Файл:** Multiple - `Button.tsx`, `LineCard.tsx`, `PrepareStep.tsx`  
**Проблема:** Стандарт WCAG/iOS/Android требует минимум 44x44px для touch targets

| Компонент | Размер | Стандарт | Статус |
|-----------|--------|----------|--------|
| Button | py-2.5 (~32-36px) | 44px | ❌ FAIL |
| Eye icon | h-9 w-9 (36px) | 44px | ❌ FAIL |
| Delete icon | h-9 w-9 (36px) | 44px | ❌ FAIL |
| File buttons | h-7 w-7 (28px) | 44px | ❌❌ FAIL |
| Dropdowns | py-2 + text (~28-32px) | 44px | ❌ FAIL |

**Видимое поведение:**
- На мобиле сложно нажимать на кнопки/иконки
- Неточные клики, пользователь нажимает соседний элемент
- Особенно плохо на планшетах с дальним держанием

**Решение:**
- Все интерактивные элементы на мобиле: минимум `h-11 w-11` (44px)
- Desktop может быть меньше (32px OK)

**Code fix:**
```html
<!-- На мобиле -->
<button className="h-11 w-11 md:h-9 md:w-9 ...">
  <Icon size={20} />
</button>
```

**Impact:** 🟡 **ВЫСОКОЕ** - пользователи делают ошибки, low precision

### 7. Таблица горизонтального скролла на tablet
**Файл:** `LinesTable.tsx`  
**Строки:** 57  
**Проблема:**
```html
<table className="hidden w-full min-w-[620px] ... md:table">
```
- `min-w-[620px]` на md (768px) создаёт горизонтальный scroll
- Должно быть либо скрыто до lg, либо иметь responsive вариант

**Решение:**
- Скрыть до lg: `hidden lg:table`
- Или добавить sm variant с меньшей шириной

**Impact:** 🟡 **СРЕДНЕЕ** - tablet users (640-768px) видят horizontal scroll

## 🟠 СРЕДНИЙ ПРИОРИТЕТ (P2)

### 8. ValidationPanel исчезает при скроллинге
**Файл:** `InvoiceWorkbench.tsx`  
**Строки:** 509-519  
**Проблема:**
- ValidationPanel показывает блокирующие ошибки (красный) и предупреждения (оранжевый)
- При скроллинге к таблице для маппинга SKU панель уходит из видимости
- Пользователь должен скроллить вверх, чтобы узнать, что не так

**Решение:**
- Сделать sticky на top: `sticky top-0`
- Или показывать компактную версию в mobile action bar
- Или collapse в compact mode на мобиле

**Impact:** 🟠 **СРЕДНЕЕ** - требует extra scrolling, но не блокирует

---

### 9. Photo accordion требует дополнительного клика
**Файл:** `InvoiceWorkbench.tsx`  
**Проблема:**
- На мобиле: кнопка "Накладная" отдельно от таблицы
- Нужно два клика: открыть фото, затем закрыть и видеть таблицу
- Photo context-switching breaks focus

**Решение:**
- Мобиль: убрать фото совсем или сделать inline-expandable
- Desktop: OK текущее поведение

**Impact:** 🟠 **СРЕДНЕЕ** - friction, но есть workaround

---

### 10. Photo button может переноситься на узких экранах
**Файл:** `InvoiceWorkbench.tsx`  
**Строки:** 505-510  
**Проблема:**
```html
<button className="inline-flex ... ">
  {photoVisible ? '▼' : '▶'} Накладная
</button>
```
- На очень узких экранах (< 280px) кнопка может wrap
- Иконка и текст могут быть на разных строках

**Решение:**
- Убрать text на мобиле или сделать иконку-only
- `<span className="hidden sm:inline">Накладная</span>`

**Impact:** 🟠 **НИЗКОЕ** - cosmetic, не влияет на функциональность

## 📝 Чек-лист для QA

При тестировании исправлений проверить:

### Мобиль (320px, 375px, 414px)
- [ ] Таблица прокручивается вниз (видны все строки)
- [ ] Кнопка "Добавить строку" видна, не закрыта полосой
- [ ] Все поля формы вмещаются в ширину без горизонтального скролла
- [ ] Поля стэкуются вертикально (не flex-wrap)
- [ ] Все кнопки/иконки минимум 44x44px
- [ ] Фото показывает детали (400px+ высоты)

### Tablet (640px, 768px, 1024px)
- [ ] Форма читается комфортно (max-width или grid-cols-2)
- [ ] Таблица НЕ создаёт горизонтальный scroll на 640-768px
- [ ] Touch targets достаточные (≥ 44px)
- [ ] Transition от мобиля к десктопу гладкий

### Desktop (1200px+)
- [ ] Таблица полностью видна с scroll внутри контейнера
- [ ] Photo accordion позиционирован правильно
- [ ] Validation panel видна всегда (sticky или всегда в фокусе)
- [ ] No horizontal scroll
