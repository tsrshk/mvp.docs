# Local OS

**AI-помощник для владельца малой кофейни.** Снимает с владельца рутинную и аналитическую работу — от ввода накладных до сравнения поставщиков и анализа конкурентов района.

---

## Идея

Владелец малой кофейни тратит десятки часов в месяц на однотипные операции, которые требуют внимания, но не приносят радости: перенос накладных в учётную систему, поиск «у кого лучше заказать», запоминание чем мы отличаемся от соседей. Существующие POS-системы (Esupl, iiko, Poster) хорошо считают деньги внутри бизнеса, но не помогают **думать** о бизнесе.

Local OS встраивается между жизнью кофейни и её учётной системой, забирает на себя рутинные операции и даёт владельцу думающего партнёра, который анализирует данные, ищет альтернативы, сравнивает с конкурентами и помогает принимать решения.

Принцип: **AI собирает данные и показывает аргументы — человек принимает решения**. Никаких автоматических заказов, никаких «AI решил за тебя». Это инструмент мышления, а не замена владельца.

---

## Что решает

Четыре конкретные боли владельца малой кофейни:

1. **Ручной ввод накладных в учётную систему** — самая частая и времязатратная рутина (десятки накладных в месяц)
2. **Разные названия одного товара у разных поставщиков** — «Зерно эфиопия Иргачеф 1кг» = «Эфиоп Йиргачефф 1000г» = один и тот же SKU в Esupl
3. **«У кого лучше заказать?»** — решения о выборе поставщика на основе привычки и переписки в WhatsApp, а не сравнения данных
4. **Позиционирование относительно района** — владелец не знает что у соседей с ценами, ассортиментом, отзывами

---

## Как работает (коротко)

Local OS подключается к учётной системе кофейни (Esupl) и накапливает данные: продажи, остатки, рецепты, поставщиков, цены. Параллельно собирает внешний контекст — отзывы Google Maps, меню конкурентов района, погоду, локальные события. Поверх данных работает AI-слой на Claude API: распознаёт накладные с фото, мапит названия позиций, сравнивает цены, анализирует тренды отзывов, формирует еженедельные дайджесты.

Взаимодействие — через веб-интерфейс. AI инициирует разговоры по расписанию (понедельничные сводки, алёрты по аномалиям) и отвечает на вопросы владельца в свободной форме.

## Функциональные возможности

Разработка идёт **по стадиям**, каждая стадия — самостоятельный работающий инструмент. Не «всё или ничего», а постепенное наращивание.

### Фаза 1 — для собственной кофейни

| Стадия | Что делает |
|---|---|
| **F1 · OCR накладных** | Фото накладной → структурированные данные (поставщик, позиции, количество, цена). Распознавание через Claude Vision. |
| **F2 · Маппинг и приход в Esupl** | Каждая позиция мапится на SKU в Esupl. Связь запоминается — следующая накладная от того же поставщика мапится автоматически. После подтверждения — приход записывается в Esupl через API. |
| **F3 · Справочник поставщиков** | Персональная база поставщиков и их цен. Цены обновляются автоматически при каждой новой накладной. История изменений. |
| **F4 · Сравнение поставщиков** | «Зерно эфиопии у текущего поставщика 1350, у альтернативного 1380 с мин. партией 5кг.» Проактивные алёрты при росте цен. |
| **F5 · Аналитика продаж** | Еженедельный дайджест: выручка, маржа, аномалии, топ-позиции. На основе данных из Esupl. |
| **F6 · Локальный контекст** | Учёт погоды и событий района в прогнозах и объяснениях аномалий. |
| **F7 · Конкуренты: меню и цены** | База цен 5–10 ближайших кофеен (по фото меню). Сравнение «у вас vs у района». |
| **F8 · Конкуренты: отзывы** | AI-анализ отзывов Google Maps: тренды, sentiment, упоминания позиций. |
| **F9 · Cross-recipe рекомендации** | «Из ваших ингредиентов можно сделать ещё 7 позиций без больших затрат.» |
| **F10 · Стратегические инсайты** | Еженедельный разговор с AI: 3 главных вопроса недели + диалог в свободной форме. |

После F10 — **Wife-Gate**: критерий перехода в фазу 2. Продукт должен стать необходимой частью рабочей недели владельца.

### Фаза 2 — для рынка

Когда продукт работает для одной кофейни и приносит измеримую ценность — упаковываем для других владельцев малых кофеен. Multi-tenant, биллинг, self-service onboarding, web-кабинет, поддержка iiko в дополнение к Esupl. Цель — выйти на $1M годовой прибыли через 5 лет от подписок $99–149/мес.

---

## Что НЕ делает

Чтобы границы продукта были чёткими:

- Не оформляет заказы автоматически — владелец пишет поставщику сам
- Не действует в Esupl без подтверждения человека
- Не парсит соцсети (Instagram, ВК, TikTok) — серая зона, шум
- Не делает финансовую отчётность, расчёт зарплат, учёт смен — это в Esupl
- Не агрегирует цены между разными клиентами (приватность)
- Не работает offline

## Технологии

| Слой | Решение |
|---|---|
| Backend | Python 3.12 + FastAPI + SQLAlchemy 2.0 async |
| Frontend | React + Vite + TypeScript + Tailwind v4 + RTK Query + FSD |
| База данных | PostgreSQL 16 + pgvector |
| AI | Claude API (текст, Vision для OCR), Voyage embeddings |
| Инфраструктура | Docker Compose локально → Hetzner на фазе 2 |
| Интеграции | Esupl API, Google Maps Platform, OpenWeather |

Архитектурный принцип: **multi-tenant ready, single-tenant first**. Все базовые механизмы (модули, тенанты, аудит) заложены с первого дня, реализованы минимально — расширение в фазу 2 без переписывания.

## Команда и ресурсы

| Кто | Роль |
|---|---|
| Founder | Разработка (frontend, продукт, AI). ~10–15 ч/нед вечерами. |
| Жена | Customer Zero, владелец кофейни. Источник требований и ежедневного фидбека. |
| Java BE парт-тайм | Подключается на фазе 2 (multi-tenant, биллинг). На фазе 1 не нужен — Python достаточно. |

**Бюджет фазы 1:** $500–1500 (LLM API + хостинг данных + инфраструктура). На фазе 2 — самофинансирование из MRR.

---

## Метрики успеха

**Фаза 1:** жена сама говорит «без этого хуже» после 4 недель использования. Измеримо сэкономлено ≥3 часа в неделю. Это и есть **Wife-Gate** — критерий перехода в фазу 2.

**Фаза 2:** 10 платящих клиентов с MRR $1K, retention 80%+, измеримая ценность для каждого. Если к месяцу 12 не достигнуто — проект закрывается без сожалений.

---

## Главный принцип

> **«Сначала помочь жене. Потом — деньги.»**

Если фаза 1 не работает — ко второй не переходим. Это защищает от классической стартап-ловушки «строим продукт для рынка, никто не покупает». Customer Zero под боком, ежедневный фидбек, нулевая стоимость неправильного направления.
