# Баг-трекинг LCOS

Дефекты живут в git как Markdown, в стиле остального vault: один баг — один файл,
frontmatter + вики-ссылки, SSOT. Отслеживаемость даёт [[_INDEX]] через Dataview
(живая таблица, ничего не ведётся руками). Внешний трекер не нужен, пока команда
малая; при росте те же файлы выгружаются в Jira через `speckit-taskstoissues`.

## Тип и нумерация
- **Тип:** `bug`, ID `LCOS-B#` (grouped-typed, как `LCOS-F#`/`LCOS-E#`).
- Следующий свободный номер — снизу таблицы в [[_INDEX]].
- Файл: `LCOS-B<#>-<kebab-slug>.md`, шаблон — [[_TEMPLATE]].

## Жизненный цикл (минимальный, для соло/1-2 чел.)

```
open ──▶ in-progress ──▶ fixed ──▶ verified
  │                                    
  ├──▶ wontfix    (не чиним осознанно)
  └──▶ dup         (дубль, ссылка на оригинал в поле `dup_of`)
```

| status | смысл |
|---|---|
| `open` | заведён, ждёт разбора/работы |
| `in-progress` | в работе |
| `fixed` | фикс в коде, ждёт проверки (проставь `commit`) |
| `verified` | проверено на реальном сценарии (проставь `verified`) — баг закрыт |
| `wontfix` | сознательно не чиним (причина в теле) |
| `dup` | дубликат, см. `dup_of: "[[LCOS-B#-…]]"` |

## Severity
| | |
|---|---|
| `S1` | блокер: падение, потеря данных, сценарий недоступен |
| `S2` | серьёзный: ключевой путь ломается, обход дорогой |
| `S3` | незначительный: есть простой обход |
| `S4` | косметика/мелочь |

## Обязательные поля frontmatter
`id`, `type: bug`, `title`, `status`, `severity`, `area` (`backend`/`frontend`/`infra`/`docs`),
`found` (дата), `owner`. Ссылки: `feature: "[[LCOS-F…]]"` (какую фичу задевает —
это и есть связь с SSOT), опц. `entities`, `requirements`, `adrs`. При фиксе — `commit`,
`fixed`; при проверке — `verified`.

## Связь со spec-kit
Баг **ссылается на фичу** через `feature:`. Speckit остаётся build-потоком:
- мелкий фикс → правишь код + ставишь `verified`, при необходимости обновляешь AC во
  фиче (SSOT побеждает — см. [[README]] порядок доверия);
- баг вскрывает пробел в спеке → заводишь/уточняешь фичу через `/speckit-specify`
  и линкуешь баг в её `sources`;
- при переезде в Jira — `speckit-taskstoissues` выгружает эти же записи.

Обратная ссылка: в задетой фиче можно добавить `[[LCOS-B#-…]]` в раздел заметок,
чтобы связь была двусторонней.
