# Фаза F7 — Конкуренты: меню и цены

> Боль: «а сколько у соседей?» — цены района неизвестны. Продукт: справочник 5–10 ближайших
> кофеен; фото их меню при обходе → OCR (переиспользование инфраструктуры F1/S2) → база цен
> конкурентов; сравнение «у нас vs район». Сбор данных — только легальный ручной обход
> с фото; автопарсинг сайтов НЕ делается (серая зона, из исходной стадии).
> Сквозные требования — `00_IMPLEMENTATION_PLAN.md` §4.

**Цель:** база покрывает 5+ конкурентов и 80% наших топ-позиций; 1–3 обоснованных решения
по ценам/меню за квартал.

**Зависимости:** S2 (OCR-инфраструктура: multi-page, кроп). **Оценка:** 3 недели.

---

## 1. Backend

### F7-B1. Модель данных
- `competitors` (`OrganizationScopedMixin`, uuid pk): `name`, `address?`, `lat?/lon?`,
  `kind?` (кофейня/пекарня/...), `rating?`, `google_place_id?`, `note?`, `is_active`.
- `competitor_menu_snapshots` (`OrganizationScopedMixin`, uuid pk): `competitor_id`
  (FK CASCADE), `captured_on` (date), `image_ref?`, `ocr_raw? Text` — один обход = один снапшот.
- `competitor_menu_items` (`OrganizationScopedMixin`, int pk): `snapshot_id` (FK CASCADE),
  `name` (как в меню), `canonical_name?` (наша нормализация, напр. «капучино 300»),
  `price Numeric(14,2)`, `currency`, `volume?` (str), `category?`.
- «Текущее меню конкурента» = позиции последнего снапшота; история сравнений — по снапшотам.

### F7-B2. OCR меню (переиспользование, новый doc-type)
- Расширить OCR-контур типом документа `menu` (по образцу `invoiceType` из S2-B2):
  отдельный промпт «извлеки позиции меню: название, объём, цена, категория; JSON;
  null если не уверен». Возврат — DTO `MenuDraft` (домен).
- `POST /api/v1/competitors/{id}/menu/recognize` (multipart, до 3 фото — страницы меню) →
  `MenuDraft` (без персиста, как invoices/recognize);
  `POST /api/v1/competitors/{id}/menu` — сохранить подтверждённый снапшот с позициями.
- Всё через `ai_complete`/OcrProvider-инфраструктуру (VPN fail-closed, ключ platform).

### F7-B3. Сравнение с районом
- `GET /api/v1/competitors/compare?position=<canonical_name>`: наша цена (источник: Esupl
  menu через read-провайдер F5, если доступно; иначе ручное поле «наша цена» на позиции
  сравнения), средняя/мин/макс района, по-конкурентно.
- `canonical_name` — назначается при подтверждении снапшота: fuzzy-подсказка
  (переиспользовать `rankSkus`-подход на бэке или простой Jaccard) + ручное подтверждение.
  Немапленные позиции остаются с `canonical_name=NULL` и в сравнение не входят.
- Напоминание «обновить базу» — раз в квартал: генерируется alert (механизм F4) для
  конкурентов с `captured_on` старше `competitor_staleness_days` (REGISTRY, default 90).
- Модульный гейт: `module_competitors_enabled`.

### F7-B4. (Опционально) Google Places для первичного списка
- Только если бесплатного тарифа достаточно: ключ в
  `integration_credentials(platform, google_places)`, поиск кофеен в радиусе → предзаполнение
  карточек (имя/адрес/рейтинг). Недоступно/нет ключа → ручное создание карточек (основной путь).

## 2. Frontend

### F7-F1. Раздел «Конкуренты»
- Список конкурентов (карточки: имя, адрес, дата последнего меню, бейдж «устарело»);
  создание/редактирование карточки.
- Экран конкурента: текущее меню (позиции/цены), кнопка «Обновить меню» → поток
  фото → кроп → распознавание → ревью-таблица (редактирование позиций, маппинг
  canonical_name с подсказками) → сохранить снапшот. Максимальное переиспользование
  компонентов workbench/prepare-step (фото-конвейер S2).
- Экран сравнения: выбор позиции → «у нас 5.50 · район: средняя 5.30 · 4.80–6.20» +
  таблица по конкурентам. Мобайл-фёрст.
- mock-провайдер: демо-конкуренты и меню.

## 3. Вне объёма
- Автопарсинг сайтов/агрегаторов конкурентов; отзывы (F8); автоматическое изменение
  наших цен (только информация — G7).

## 4. Acceptance Criteria

- [ ] AC-1. CRUD конкурента из UI; тенант-изоляция таблиц фазы покрыта тестом.
- [ ] AC-2. Фото меню (1–3 страницы) → распознанные позиции в ревью-таблице → правка →
      снапшот сохранён; повторное обновление меню создаёт НОВЫЙ снапшот, старый читаем
      (интеграционный тест + браузерная верификация с реальным фото меню).
- [ ] AC-3. OCR меню использует общий AI-контур: без ключа → `AiUnavailableError` 503;
      с `ai_vpn_enabled=true` и мёртвым туннелем → `VpnUnavailableError` (тесты).
- [ ] AC-4. Маппинг canonical_name: подсказка предлагается, ручной выбор персистится;
      позиции без canonical_name не попадают в сравнение (тест).
- [ ] AC-5. Сравнение по позиции считает среднюю/мин/макс только по последним снапшотам
      активных конкурентов (unit-тест с 2 снапшотами разных дат).
- [ ] AC-6. Конкурент с меню старше порога получает квартальный alert «пора обновить базу»;
      порог из SQLAdmin (тест).
- [ ] AC-7. `module_competitors_enabled=false` → 404 + скрытый раздел UI.
- [ ] AC-8. Метрика: занесено ≥5 реальных конкурентов, ≥80% наших топ-позиций имеют
      сравнение (фиксация в `work/phase-f7.md` после реального обхода).
- [ ] AC-9. Общий DoD G10.
