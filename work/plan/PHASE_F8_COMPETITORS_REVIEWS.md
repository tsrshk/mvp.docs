# Фаза F8 — Конкуренты: отзывы

> Боль: что говорят о нас и о соседях — неизвестно, реакция на жалобы запаздывает.
> Продукт: хранение и AI-анализ отзывов (sentiment, тренды жалоб, упоминания позиций),
> секция «Отзывы» в дайджесте. **Юридическое ограничение (план §6 Q5):** MVP работает
> на официально доступных данных — свои отзывы через Google Business Profile
> (API/выгрузка), отзывы конкурентов — ручной импорт; автоскрейпинг Google Maps НЕ делается.
> Сквозные требования — `00_IMPLEMENTATION_PLAN.md` §4.

**Цель:** 100% отзывов нашей кофейни обработаны в течение 48 ч; ≥1 практическое решение
в месяц на основе анализа.

**Зависимости:** F7 (карточки конкурентов), F5 (дайджест — для секции; при отсутствии F5
секция откладывается). **Оценка:** 2 недели.

---

## 1. Backend

### F8-B1. Модель данных
- `reviews` (`OrganizationScopedMixin`, uuid pk): `subject` (enum: `self` | `competitor`),
  `competitor_id?` (FK, NULL для своих), `source` (enum: `google` | `manual` | `import`),
  `external_id?` (unique в org — дедупликация), `author?`, `rating?` (smallint),
  `text` (Text), `posted_at` (date/tz), `lang?`.
- `review_analyses` (uuid pk, FK→reviews CASCADE, 1:1): `sentiment`
  (enum: `positive|neutral|negative`), `topics JSONB` (список категорий: сервис, музыка,
  кофе, десерты…), `mentions JSONB` (упомянутые позиции меню), `model`, `analyzed_at`.
- Анализ отделён от отзыва: переанализ новой моделью не трогает исходник.

### F8-B2. Ingestion
- `POST /api/v1/reviews/import` — пакетный импорт (JSON-массив; для конкурентов —
  скопированные вручную/выгруженные отзывы), дедуп по `external_id`/хэшу текста.
- Свои отзывы: провайдер `reviews` за Protocol (одна реализация):
  - если Google Business Profile API доступен (OAuth кофейни настроен) —
    периодический fetch своих отзывов (джоба планировщика F5,
    `reviews_sync_enabled` REGISTRY default False; креды —
    `integration_credentials(scope=org, provider=google_business)`, новое enum-значение);
  - если нет — ручной импорт тем же `/reviews/import` (основной путь MVP; API-провайдер
    может быть отложен решением владельца при старте фазы — тогда оформить в журнале).

### F8-B3. AI-анализ
- `ReviewAnalysisService`: батч непроанализированных отзывов → `ai_complete`
  (дешёвая модель из REGISTRY) → sentiment/topics/mentions строго из фиксированного
  JSON-контракта; невалидный ответ → отзыв остаётся непроанализированным (retry при
  следующем батче), не мусор в БД.
- Запуск: после импорта + джобой раз в сутки. Бюджет: батчить по N отзывов за вызов
  (`reviews_batch_size` REGISTRY, default 20) — G11.
- Тренды (детерминированно, без LLM): агрегация topics/sentiment по неделям;
  «жалобы на X выросли» = число negative с топиком X за 4 недели vs предыдущие 4.

### F8-B4. API + дайджест
- `GET /api/v1/reviews?subject=&competitor_id=&sentiment=` (пагинация);
  `GET /api/v1/reviews/trends?weeks=8`.
- Секция «Отзывы» в metrics дайджеста (F5): новые отзывы за неделю (свои: количество,
  средний рейтинг, негативные — списком), тренды, упоминания позиций у конкурентов.
- Негативный СВОЙ отзыв → alert (механизм F4) сразу при импорте/анализе — не ждать дайджеста
  (цель «реакция <48 ч»).
- Гейт: `module_competitors_enabled` (общий с F7) — или отдельный `module_reviews_enabled`,
  решение исполнителя с фиксацией.

## 2. Frontend

### F8-F1. Вкладка «Отзывы» в разделе Конкуренты
- Свои/конкурент: список отзывов с sentiment-бейджами и фильтрами; экран трендов
  (простые полосы/счётчики по неделям, без тяжёлых chart-библиотек).
- Импорт: форма вставки текста/JSON (ручной путь) с предпросмотром распознанных записей.
- Алерты о негативных отзывах — в общем механизме алертов.

## 3. Вне объёма
- Автоответы на отзывы (никогда без человека — G7); соцсети (Instagram/VK/TikTok — не
  делается, из About); скрейпинг Google Maps конкурентов.

## 4. Acceptance Criteria

- [ ] AC-1. Импорт пакета из 10 отзывов: все сохранены, повторный импорт того же пакета
      не создаёт дублей (тест).
- [ ] AC-2. Анализ батча (respx-мок ai_complete): sentiment/topics/mentions сохранены;
      невалидный JSON от LLM → отзыв остаётся в очереди, ошибки нет (тест).
- [ ] AC-3. Негативный отзыв о СВОЕЙ кофейне порождает alert при обработке (тест);
      в UI виден с бейджем негатива.
- [ ] AC-4. Тренды считаются детерминированно (unit-тест: фикстура 8 недель → рост жалоб
      на «музыку» детектируется).
- [ ] AC-5. Секция «Отзывы» появляется в очередном дайджесте при наличии данных
      (интеграционный тест генерации дайджеста).
- [ ] AC-6. AI недоступен → импорт работает, анализ откладывается с явным статусом
      (не тихо); никаких LLM-вызовов вне батчей по расписанию/после импорта (G11).
- [ ] AC-7. Если реализован Google-fetch: креды `scope=org` в `enc:v2:*`, синк идемпотентен,
      без кредов — fail-closed ошибка sync_run (тест). Если отложен — ADR/журнал.
- [ ] AC-8. Тенант-изоляция reviews покрыта тестом; модульный гейт работает.
- [ ] AC-9. Реальная проверка: свежие отзывы своей кофейни импортированы и обработаны
      <48 ч от появления (фиксация в `work/phase-f8.md`).
- [ ] AC-10. Общий DoD G10.
