# Фаза F9 — Cross-recipe рекомендации

> Боль: «что ещё добавить в меню без больших затрат?» — гипотезы и пробы. Продукт:
> AI-генерация идей позиций из имеющихся SKU с экономическим обоснованием (что уже есть,
> что докупить и почём — из F3/F4, бенчмарк цены — из F7). Человек решает: идея ≠ действие
> (G7); рецепт финализируется дегустацией.
> Сквозные требования — `00_IMPLEMENTATION_PLAN.md` §4.

**Цель:** ≥3 идеи в месяц с обоснованием; ≥1 реально добавленная позиция в квартал.

**Зависимости:** F3 (цены SKU); желательно F7 (бенчмарки цен района — если нет, поле
benchmark пустое). **Оценка:** 2 недели.

---

## 1. Backend

### F9-B1. Генерация идей
- `POST /api/v1/menu-ideas/generate` (по кнопке пользователя; опционально `season` /
  `focus` из запроса) → `MenuIdeaService`:
  1. Собирает контекст ДЕТЕРМИНИРОВАННО: локальный каталог SKU (`ingredients` + фасовки),
     текущие цены (F3, последние, не stale), топ-продажи (F5, если есть), позиции
     конкурентов (F7, если есть).
  2. Один вызов `ai_complete` (основная модель) с жёстким JSON-контрактом:
     `ideas[{title, description, uses_skus[{ingredient_id, qty_note}],
     missing_ingredients[{name, est_price_note}], selling_price_suggestion, rationale}]`.
  3. Пост-валидация: `uses_skus.ingredient_id` обязаны существовать в каталоге —
     идеи с выдуманными SKU отбрасываются целиком (анти-галлюцинация, как `_parse_matches`).
  4. Себестоимость по имеющимся SKU досчитывается кодом из цен F3 (не доверять цифрам LLM);
     цены LLM допустимы только в `est_price_note` для отсутствующих ингредиентов,
     помечены как оценка.
- Хранение: `menu_ideas` (`OrganizationScopedMixin`, uuid pk): `title`, `description`,
  `payload JSONB` (полная идея + расчёт), `status` (enum: `new|liked|dismissed|adopted`),
  `season?`, `created_at`, `generated_by_model`.
- Лимит: не более `menu_ideas_daily_limit` генераций в сутки (REGISTRY, default 3) — G11.

### F9-B2. API
- `GET /api/v1/menu-ideas?status=`, `PATCH /api/v1/menu-ideas/{id}` (смена статуса).
- Гейт: `module_menu_ideas_enabled`.

## 2. Frontend

### F9-F1. Страница «Идеи меню»
- Кнопка «Предложить идеи» (+ выбор сезона/фокуса), карточки идей: состав
  (есть ✓ / докупить + оценка), расчёт себестоимости, предлагаемая цена, аргументация,
  бенчмарк района (если есть F7).
- Действия: 👍 (liked) / скрыть (dismissed) / «добавили в меню» (adopted) — статусы
  питают метрику стадии.
- mock-провайдер — демо-идеи.

## 3. Вне объёма
- Автосоздание рецептов/техкарт в Esupl (никакой записи — G6); тренды соцсетей (Phase 2+);
  автоматический пересчёт идей по расписанию (только по кнопке).

## 4. Acceptance Criteria

- [ ] AC-1. Генерация (respx-мок LLM): идеи с несуществующими ingredient_id отброшены;
      себестоимость имеющихся SKU посчитана кодом из цен F3 и совпадает с ручным расчётом
      (unit-тест на Decimal).
- [ ] AC-2. Идеи персистятся; статусы меняются из UI и сохраняются (тест + браузерная
      верификация).
- [ ] AC-3. Дневной лимит генераций работает; превышение → явная ошибка с сообщением;
      лимит меняется из SQLAdmin (тест).
- [ ] AC-4. AI недоступен/VPN down → 503 с envelope, страница показывает ошибку, ранее
      сгенерированные идеи доступны (тест + ручная проверка).
- [ ] AC-5. Никаких LLM-вызовов без нажатия пользователя (grep/ревью: generate — только
      из роута).
- [ ] AC-6. Тенант-изоляция menu_ideas; модульный гейт.
- [ ] AC-7. Реальная проверка: на живом каталоге сгенерировано ≥3 осмысленных идеи
      (оценка владельцем, фиксация в `work/phase-f9.md`).
- [ ] AC-8. Общий DoD G10.
