# LCOS-F53 · Обогащение дайджеста аномалиями
**Epic:** [[LCOS-E10-local-context]] · **Status:** future · **Phase:** Phase 2

## Описание

Замыкает цикл локального контекста: для каждого аномального дня продаж, помеченного еженедельным дайджестом ([[LCOS-F48-weekly-digest]] в [[LCOS-E9-sales-analytics]]), прикрепить правдоподобную внешнюю причину из погоды ([[LCOS-F50-weather]]) и событий ([[LCOS-F52-local-events]]). Корреляция **детерминированная, не на LLM**: день аномален, когда отклоняется >±20% от 4-недельного среднего для того же дня недели, и контекст прикрепляется, когда осадки превышают порог, температура отклоняется за порог или событие пересекается с днём.

`metrics` дайджеста получают `anomalies[{date, delta_pct, context[]}]` и `upcoming[{date, kind, title}]` (события + экстремальный прогноз на предстоящую неделю). LLM-текст дайджеста (из фичи дайджеста аналитики продаж) получает только эти предвычисленные данные и может ссылаться на контекст **только** из них — та же дисциплина против галлюцинаций, что и в других местах ([[provider-abstraction]]). Пороги аномалий (`anomaly_rain_mm` default 5, `anomaly_temp_delta` default 8) и расписание настраиваются через config-реестр без редеплоя. Когда причина не найдена, дайджест сообщает «причина не найдена», а не выдумывает её.

## Возможности

- Детерминированная детекция аномалий (>±20% vs 4-недельное среднее того же дня недели) — без LLM на шаге корреляции.
- Прикрепление контекста из `weather_days` (пороги дождя/температуры) и `local_events` (пересечение).
- `metrics` дайджеста расширены `anomalies[]` и `upcoming[]` (события + экстремальный прогноз на следующую неделю).
- Против галлюцинаций: LLM-текст дайджеста может ссылаться только на контекст, присутствующий в `metrics`; явное «причина не найдена» при пустоте.
- Настраиваемые в рантайме пороги через config-реестр (`anomaly_rain_mm`, `anomaly_temp_delta`).
- FE: блок аномалий («Чт −30% · дождь 12 мм»), блок «Следующая неделя» (события + прогноз), погода по дням в детальном представлении через `context/days`.
- `GET /api/v1/context/days?from=&to=` — погода + события для UI.

## Доступ по ролям

| Роль | Что может делать |
|---|---|
| [[member]] | Читает обогащённый дайджест (объяснения аномалий + предупреждения о предстоящем). |
| [[admin]] | То же; также курирует входные данные через [[LCOS-F51-coordinates]] и [[LCOS-F52-local-events]]. |
| [[superadmin]] | Настраивает пороги аномалий/расписание через config-API по тенантам. |
| [[sqladmin-operator]] | Корректирует пороги/флаги реестра в плоскости SQLAdmin (см. [[LCOS-F3-sqladmin-operator]]). |

Тенант-скоуп: обогащение читает только собственные погоду/события подразделения ([[multitenancy]]).

## Задействованные сущности

- [[subdivisions]] — скоуп дайджеста и его контекстных входов.
- [[system_settings]] — пороги аномалий и расписание, разрешаемые в рантайме.
- `weather_days`, `local_events` (будущие таблицы со скоупом подразделения) — источники контекста; документы-сущности создаются при активации.

- **Features:** обогащает [[LCOS-F48-weekly-digest]] ([[LCOS-E9-sales-analytics]]); потребляет [[LCOS-F50-weather]] и [[LCOS-F52-local-events]]; зависит от [[LCOS-F51-coordinates]] для погодного фида.
