# LCOS-F75 · Аналитика цен и ассортимента поставщика

**Эпик:** [[LCOS-E4-suppliers]] · **Статус:** built · **Фаза:** Phase 1

## Описание

Аналитический слой поверх единого временного ряда `supplier_prices`. Поскольку `supplier_prices` расширен дискриминатором `source ∈ {invoice, manual, price_list_upload}` (см. [[LCOS-F20-price-history]]), все наблюдения цен — из инвойсов, ручного ввода и загруженных прайс-листов/фото/буклетов — лежат в одном ряду. F75 читает этот единый ряд **независимо от источника** и строит две вещи: рост цены по SKU во времени и рост ассортимента (сколько уникальных предлагаемых SKU у поставщика во времени).

Это **даунстрим-фича с лёгким AC**: она не вводит новых сущностей и не пишет данные, а только агрегирует уже собранное. Реализуется **после** [[LCOS-F72-supplier-price-list-upload]], [[LCOS-F73-price-list-parsing]] и [[LCOS-F74-supplier-assortment-freshness]] — то есть после того, как прайс-листы начали проецироваться в `supplier_prices`, и поверх [[LCOS-F20-price-history]].

Родственные работы: [[LCOS-F21-price-change-signal]] (точечный сигнал изменения цены в потоке инвойсов — F75 даёт исторический разрез того же самого) и план сравнения поставщиков (`work/plan/PHASE_F4_SUPPLIER_COMPARE`). Открытый вопрос — не слить ли аналитику F75 с фичей сравнения F4.

## Возможности

- (Запланировано) Временной ряд цены за базовую единицу по паре `(supplier, SKU)` — построен из `supplier_prices`, по всем источникам `{invoice, manual, price_list_upload}`.
- (Запланировано) Метрика роста ассортимента: количество уникальных предлагаемых SKU у поставщика по датам (динамика ширины ассортимента во времени).
- (Запланировано) Чтение единого `supplier_prices` без фильтра по `source` — данные из инвойсов, ручного ввода и прайс-листов равноправны в агрегации.
- (Запланировано) Разрез в пределах scope арендатора (`organization_id`).

## Доступ по ролям

| Роль | Что может делать |
|---|---|

| [[admin]] | То же, внутри subdivision. |

Scope арендатора (`organization_id`) приходит из активного контекста JWT (см. [[multitenancy]]).

## Задействованные сущности

- [[price_list_line]] — позиции прайса (append-only), которые вместе с наблюдениями из инвойсов и ручного ввода проецируются в единый ряд `supplier_prices` и служат исходником агрегаций.

- **Требования:** [[multitenancy]] (агрегация в пределах scope арендатора).
- **Фичи:** [[LCOS-F20-price-history]] (единый ряд `supplier_prices` с дискриминатором `source` — жёсткая основа), [[LCOS-F21-price-change-signal]] (родственный сигнал изменения цены; F75 — исторический разрез), [[LCOS-F74-supplier-assortment-freshness]] (ассортимент/freshness карточки, поверх которого строится динамика ширины ассортимента).
- **План:** `work/plan/PHASE_F4_SUPPLIER_COMPARE` — сравнение поставщиков; открыт вопрос о слиянии с F75.
- **ADR:** нет специфических.

## Критерии приёмки (AC)

### Backend
- [ ] AC-BE-1. Временной ряд цены за базовую единицу возвращается по паре `(supplier, SKU)`, построенный из `supplier_prices`.
- [ ] AC-BE-2. Метрика роста ассортимента: количество уникальных предлагаемых SKU у поставщика по датам.
- [ ] AC-BE-3. Агрегации читают единый `supplier_prices` независимо от `source` (`invoice`, `manual`, `price_list_upload` учитываются одинаково).
- [ ] AC-BE-4. Все разрезы в пределах scope арендатора (`organization_id`).

### Frontend
- [ ] AC-FE-1. Аналитика цен по `(supplier, SKU)` и динамика ширины ассортимента отображаются на карточке/аналитике поставщика (лёгкий вид, точная подача — открытый вопрос).

### Прочее
- [ ] AC-OTHER-1. Зависит от [[LCOS-F20-price-history]] и проекции прайс-листов (F72-F74): без наполненного `supplier_prices` аналитику нельзя проверить.

## Открытые вопросы / гейты

- **Слияние с F4-сравнением:** не объединить ли аналитику F75 с фичей сравнения поставщиков (`work/plan/PHASE_F4_SUPPLIER_COMPARE`) — открытое продуктовое решение.
- **Даунстрим-гейт:** реализуется после F72-F74 и поверх F20; до наполнения единого `supplier_prices` фича заблокирована.
- **Глубина метрик:** набор метрик намеренно лёгкий (рост цены по SKU + рост ассортимента); расширение — по мере запросов пилота.
