# LCOS-F72 · Загрузка прайс-листа поставщика

**Эпик:** [[LCOS-E4-suppliers]] · **Статус:** planned · **Фаза:** Phase 1

## Описание

F72 даёт внутреннему пользователю загрузить прайс-лист поставщика в любой форме входа — файл (Excel/CSV), фото, буклет или текстовое сообщение — и сохранить исходный документ вместе с метаданными как отдельную версию прайса на пару `(organization_id, supplier_id)`.

Фича отвечает только за приём и хранение: она создаёт запись `price_list_upload` (форма входа, MIME-тип, имя файла, ссылка на сохранённый исходник `storage_ref` или вставленный `raw_text`, дата действия прайса, монотонная версия, статус `uploaded`) и **предоставляет эндпоинт запуска парсинга кнопкой** (`POST /price-lists/{id}/parse`). Парсинг НЕ авто-триггерится на загрузке — он запускается отдельным синхронным вызовом (Phase 1 без шедулера, см. [[ARCH-SUPPLIER-PRICELISTS]] §D1). Сам разбор в структурированные `price_list_line`, OCR фото/буклета, резолв SKU и проекция в `supplier_prices` живут в [[LCOS-F73-price-list-parsing]].

Версии прайса **append-only**: каждая новая загрузка для того же поставщика получает `version = N+1`, прежние версии не перезаписываются, что сохраняет историю поступлений прайсов. Загрузка сразу видна со статусом `uploaded` и далее переходит по жизненному циклу парсинга.

Модуль поставщиков и его локальный SSOT (список/карточка из локальной таблицы `suppliers`) фиксируются в [[ADR-021]], который реверсирует часть DEC-0011 только для справочника поставщиков.

## Возможности

- Multipart-загрузка: файл (Excel/CSV/PDF) **или** фото/буклет **или** `raw_text` для входа-сообщения; форма входа фиксируется в `source_kind` (`price_file | photo | booklet | message`).
- Хранение исходника: `storage_ref` (ключ/путь к сохранённому файлу/фото), `mime_type`, `original_filename`; для сообщения — `raw_text`.
- Дата действия прайса `effective_date` из документа или введённая вручную; при NULL freshness берёт дату загрузки.
- Монотонная версия `version` на пару `(organization_id, supplier_id)`; загрузки **append-only** — прежние версии не перезаписываются.
- Запуск парсинга кнопкой (`POST /price-lists/{id}/parse`, синхронно) — НЕ авто-триггер; статус загрузки отражает жизненный цикл (`uploaded | parsing | parsed | needs_review | failed`).
- Тенант-изоляция по `organization_id`; вся фича за гейтом модуля поставщиков.

## Критерии приёмки (AC)

### Backend
- [ ] AC-BE-1. Миграция создаёт `price_list_uploads` со всеми полями канона (изоляция арендатора по `organization_id`); тест: org-A не видит загрузки org-B.
- [ ] AC-BE-2. `POST /suppliers/{id}/price-lists` принимает multipart-файл, фото **или** `raw_text`-сообщение (+ optional `effective_date`), создаёт запись с `version = N+1` на пару `(organization_id, supplier_id)` и статусом `uploaded`.
- [ ] AC-BE-3. `GET /suppliers/{id}/price-lists` возвращает список версий прайса поставщика.
- [ ] AC-BE-4. `GET /price-lists/{upload_id}` возвращает одну загрузку со статусом.
- [ ] AC-BE-5. Загрузка создаётся со статусом `uploaded`; парсинг запускается ОТДЕЛЬНЫМ эндпоинтом `POST /price-lists/{id}/parse` (кнопка), НЕ авто-триггером на загрузке (Phase 1 без шедулера, [[ARCH-SUPPLIER-PRICELISTS]] §D1/§4).
- [ ] AC-BE-6. Все эндпоинты за гейтом `module_suppliers_enabled` → `404` при выключенном модуле.

### Frontend
- [ ] AC-FE-1. На карточке поставщика — блок «Прайсы» с загрузкой файла/фото и вставкой текста.
- [ ] AC-FE-2. Блок «Прайсы» показывает список версий с датами.

### Прочее
- [ ] AC-OTHER-1. Метрика: прайс пилотного поставщика загружается и версия видна в карточке.

## Открытые вопросы / гейты

- **Гейт модуля** — вся фича доступна только при включённом `module_suppliers_enabled`.
- **Форма входа фото/буклет** — самый сложный вход; приём в скоупе F72, качество разбора — ответственность [[LCOS-F73-price-list-parsing]] (мягкая деградация, `needs_review`).
