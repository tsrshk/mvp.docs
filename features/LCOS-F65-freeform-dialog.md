# LCOS-F65 · Свободный стратегический диалог
**Epic:** [[LCOS-E14-strategic-insights]] · **Status:** future · **Phase:** Phase 2

## Описание

Продолжение еженедельной сессии в открытый разговор: после трёх вопросов ([[LCOS-F64-weekly-questions]]) владелец может свободно обсуждать бизнес. Это первая по-настоящему чат-подобная поверхность в продукте — намеренно закрытая за этой ступенью лестницы рутин стоп-листом разработки стратегии (никакого чат-UI до этого шага).

Каждый ход хранится в будущей таблице `insight_messages` (uuid pk, FK → `insight_sessions` CASCADE) с `role` (`user|assistant`), `content`, `created_at`. `POST /api/v1/insights/{session_id}/messages` берёт сообщение пользователя, вызывает LLM с `(context_snapshot + история сессии + новое сообщение)`, затем сохраняет и возвращает ответ ассистента. Замороженный `context_snapshot` держит всю ветку обоснованной на тех же аудированных данных, из которых были выведены вопросы.

Ограничители: история сессии подрезается под токен-бюджет (старшие ходы суммируются или отбрасываются с маркером), применяется суточный кап `insight_dialog_daily_limit` (реестр, default 50) (защита стоимости/злоупотреблений в стиле G11), а моделирование «что если» допустимо **только** на числах, присутствующих в контексте, при этом промпт обязан показывать свою арифметику. AI аргументирует и моделирует; решает человек (G7). Чат-UI сидит под еженедельными вопросами как обычный request/response с индикатором «думает» — стриминг в Phase 2 не требуется. Явно вне объёма: авто-действия из разговора (никогда), голос, push сверх PWA и кросс-организационные бенчмарки (приватность).

## Возможности

- Диалог по ходам, сохраняемый в `insight_messages` (FK → `insight_sessions`, CASCADE); история восстанавливается при перезагрузке.
- `POST /insights/{id}/messages`: сообщение пользователя → LLM с `context_snapshot` + история сессии + новое сообщение → сохранённый ответ ассистента.
- Подрезка истории под токен-бюджет (старшие ходы суммируются/отбрасываются с маркером).
- Суточный кап сообщений `insight_dialog_daily_limit` (реестр, default 50) — защита стоимости/злоупотреблений (G11).
- Моделирование «что если» ограничено числами контекста; промпт обязан показать расчёт. AI моделирует, человек решает (G7).
- Request/response чат под еженедельной карточкой с индикатором «думает»; стриминг отложен; mock-провайдер для демо.

## Доступ по ролям

| Роль | Что может делать |

## Задействованные сущности

- [[subdivisions]] — тенант-скоуп; диалог наследует границу подразделения сессии.
- [[system_settings]] — AI-модель/промпт и `insight_dialog_daily_limit`, редактируемые без редеплоя.
- [[integration_credentials]] — зашифрованный Fernet AI-ключ, читаемый бэкендом; egress через VPN, никогда не экспонируется фронтенду.
- Будущая таблица `insight_messages` (эта фича) висит на `insight_sessions` из [[LCOS-F64-weekly-questions]]; ни одна из них не входит в замороженный набор сущностей.

## Зависимости / связи

- **Requirements:** [[provider-abstraction]] (LLM за швом), [[fail-closed]] + [[vpn-egress]] (AI/VPN лежит → явный 503 на messages; прошлые сессии остаются читаемыми), [[multitenancy]] (тенант-изоляция сообщений), [[config-secrets]] (суточный лимит, модель, промпт, гейт через трёхуровневый конфиг).
- **Features:** расширяет сессию, созданную [[LCOS-F64-weekly-questions]]; обе потребляют [[LCOS-F63-insight-context]]; гейтится [[LCOS-F6-module-gates]] (`module_insights_enabled`).
- **Epics / gates:** часть [[LCOS-E14-strategic-insights]]; проверка Pilot-Gate ([[ADR-003]]) ожидает ≥1 диалог, доведённый до решения.
